## Size and Position

### Minimap Container
- Width: 210 pixels
- Height: 200 pixels
- Position: Top-right corner
- Offset from right edge: 10 pixels
- Offset from top: 60 pixels (below top bar)

### SubViewport
- Width: 192 pixels (210 - 8px margins - 4px border)
- Height: 192 pixels (200 - 8px margins)

### Indicators
- Player triangle: ~10 pixels tall
- Container circles: 4 pixel radius (8px diameter)
- Exit diamond: 12 pixels (6px from center to point)
- Wall thickness: 2 pixels

## Animation Details

### Exit Point Pulse
```
Base size: 6 pixels
Pulse: sin(time * 0.005) * 0.2
Range: 80% to 120% of base size
Speed: ~0.005 radians per millisecond
Result: Smooth, gentle pulsing animation
```

### Player Position
- Updates every frame (60 FPS)
- Smoothly follows player movement
- No lag or stuttering

## Example Minimap States

### State 1: Just Started
- Player at entrance
- Small area revealed around spawn
- All containers hidden
- Exit point hidden
- Most of map is fog

### State 2: Partially Explored
- Several rooms revealed
- Some containers visible (yellow)
- A few searched containers (gray)
- Exit still hidden
- Path shows where player has been

### State 3: Found Exit
- Exit area revealed (cyan diamond visible)
- Most containers found
- Mix of yellow (unsearched) and gray (searched)
- Clear path to exit visible
- Some areas still unexplored

### State 4: Fully Looted
- All areas explored (no fog)
- Most containers gray (searched)
- Exit clearly visible
- Player can see entire ship layout
- Easy to navigate back to exit

## Integration with Game

The minimap updates in real-time:
1. **Every frame**: Player position updates
2. **On container interaction**: Container changes from yellow to gray
3. **As player moves**: Fog reveals, new rooms become visible
4. **Continuously**: Exit point pulses if visible

No user interaction needed - it's a passive navigation aid that always shows current game state.
